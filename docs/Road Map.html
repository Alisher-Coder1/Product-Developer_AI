Этап 1: Context Generation и Token Budget

Задачи этапа

Методы решения

- Парсинг Markdown-файлов, извлечение заголовков и секций.
- Для подсчёта токенов: GPT-3/4 tokenizer (cl100k_base).
- Реализация summary mode: обрезка менее важных разделов (например, детали реализации, комментарии).
- Тестирование на синтетических проектах.

Критерии завершения

  • Контекст генерируется корректно, включая все обязательные разделы.
  • Токены считаются с точностью >95% (сравнение с OpenAI токенизатором).
  • При превышении лимита создаётся сокращённая версия.

Интеграционное тестирование

  • Прогон на тестовом проекте с разными объёмами.
  • Проверка summary mode на граничных случаях.

Этап 2: Stage Engine и AI Task Engine

Задачи этапа

Методы решения

- Stage Engine: конечный автомат на JavaScript, проверка существования файлов/директорий.
- AI Task Engine: сначала на основе правил (rule-based), позже с использованием LLM (через OpenAI API).
- Guided workflow: интерактивный CLI с вводом/выбором.

Критерии завершения

  • Stage Engine корректно блокирует переход, если артефакты не готовы.
  • Task Engine разбивает задачу (например, "добавить аутентификацию") на микрозадачи.
  • `pos prompt` возвращает промпт с контекстом.

Тестирование

  • Тесты на сценарии: отсутствие файла, неверный этап.
  • Оценка качества декомпозиции экспертом.

Этап 3: Validation Engine и безопасность промптов

Задачи этапа

Методы решения

- Вызов внешних инструментов через `child_process` или их API.
- Сбор результатов в единый JSON-отчёт.
- Для безопасности: регулярные выражения, библиотека `detect-secrets`.
- Маскировка данных в промптах перед отправкой в AI.

Критерии завершения

  • Команда `pos validate` выполняет все проверки и возвращает отчёт.
  • При ошибке система предлагает варианты исправления (fix prompt).
  • Секреты в промптах обнаруживаются и маскируются.

Интеграционное тестирование

  • Прогон на проекте с намеренными ошибками.
  • Проверка обнаружения тестовых секретов.

Как пользоваться дорожной картой

  • Отмечайте выполненные задачи — прогресс этапов и общий прогресс обновляются автоматически.
  • Состояние сохраняется в localStorage вашего браузера, поэтому после закрытия страницы данные не пропадут.
  • Для сброса прогресса можно очистить localStorage или вручную снять все флажки.
  • Рекомендуется обновлять статусы ежедневно для актуальной картины.
