# Token Budget Engine

**Файл реализации:** `scripts/token-budget.js`  
**Назначение:** подсчёт токенов в тексте, контроль лимитов и автоматическое сокращение (summary mode) при превышении бюджета.  
**Этап Roadmap:** этап 1 (Context Generation и Token Budget)

---

## Входные данные

### `countTokens`

| Параметр | Тип    | Описание                            |
|----------|--------|-------------------------------------|
| `text`   | string | Исходный текст                      |
| `model`  | string | Модель для токенизации (по умолчанию `"gpt-4"`, в текущей версии не влияет) |

### `checkBudget`

| Параметр | Тип    | Описание                                               |
|----------|--------|--------------------------------------------------------|
| `text`   | string | Исходный текст                                         |
| `budget` | number | Максимальное допустимое количество токенов             |
| `options`| object | Дополнительные параметры (см. ниже)                    |

**Поля `options`:**

| Параметр          | Тип    | Значение по умолчанию | Описание                                        |
|-------------------|--------|-----------------------|-------------------------------------------------|
| `model`           | string | `"gpt-4"`             | Модель для токенизации (резерв)                 |
| `contextPriority` | object | `{}`                  | Переопределение приоритетов слоёв для summary mode |

## Выходные данные

### `countTokens`
Возвращает число – количество токенов.

### `checkBudget`
Возвращает объект:

```typescript
{
  count: number;                // исходное количество токенов
  exceeded: boolean;            // превышен ли бюджет
  summaryText: string;          // текст после сокращения (если превышен, иначе исходный)
  reductionStats: {
    original: number;           // исходное количество токенов
    final: number;              // количество токенов после сокращения
  };
}
Алгоритм
1. Подсчёт токенов
Используется библиотека gpt-tokenizer (токенизатор для моделей GPT). Функция countTokens(text) возвращает длину закодированной последовательности.

2. Проверка бюджета
Если count <= budget, возвращается исходный текст без изменений.

3. Режим сокращения (summary mode)
При превышении бюджета выполняется следующая процедура:

Разбиение на секции
Текст разбивается на секции по заголовкам второго уровня (##). Используется регулярное выражение /\n(?=##\s)/.

Определение слоя
Для каждой секции проверяется наличие подстрок, характерных для определённого слоя:

"product/" → слой "product"

"architecture/" → "architecture"

"domain/" → "domain"

"application/" → "application"

"contracts/" → "contracts"

"blocks/" → "blocks"
Если ни одно не найдено — слой "other".

Назначение приоритета
По умолчанию приоритеты:

product: 10

architecture: 9

domain: 8

application: 7

contracts: 6

blocks: 5

other: 1
Приоритеты могут быть переопределены через options.contextPriority.

Сортировка секций по убыванию приоритета.

Формирование итогового текста
Проход по отсортированным секциям: если текущая секция помещается в оставшийся бюджет, она добавляется. Процесс продолжается, пока бюджет не будет исчерпан или секции не закончатся.

Fallback
Если ни одна секция не помещается (например, бюджет меньше самой маленькой секции), выбирается самая приоритетная секция и включается целиком. В консоль выдаётся предупреждение.

Возврат сокращённого текста и статистики.

Пример использования
javascript
import { checkBudget } from './scripts/token-budget.js';

const longContext = `
## product/vision.md
...

## architecture/system-overview.md
...
`;

const result = checkBudget(longContext, 500, {
  contextPriority: { product: 20, architecture: 15 } // повысим приоритет product
});

if (result.exceeded) {
  console.log(`Исходный размер: ${result.reductionStats.original} токенов`);
  console.log(`После сокращения: ${result.reductionStats.final} токенов`);
  // использовать result.summaryText
} else {
  console.log('Контекст укладывается в бюджет');
}
Конфигурация
Движок не требует отдельного конфигурационного файла. Все настройки передаются через параметры функций. Приоритеты слоёв могут быть переопределены через options.contextPriority при вызове checkBudget.

Обработка ошибок
Ситуация	Реакция
budget отрицательный или не число	Исключение: "Budget must be a non-negative number"
Ошибка в библиотеке токенизации	Исключение пробрасывается из gpt-tokenizer
Нет ни одной секции (пустой текст)	Возвращается исходный текст (пустой), статистика: original = 0, final = 0
Fallback при слишком малом бюджете	Предупреждение в консоль, возвращается самая приоритетная секция
Все исключения должны обрабатываться вызывающим кодом (например, в CLI).

Связь с другими движками
AI Task Engine – использует checkBudget для оценки количества токенов в микрозадачах.

Context Generation Engine – использует countTokens для подсчёта токенов в сгенерированном контексте.
